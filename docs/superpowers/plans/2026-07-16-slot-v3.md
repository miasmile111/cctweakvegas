# Slot v3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the slot screen from the owner's mockup (15×24) and add three diegetic selectable stakes ($10/$25/$100).

**Architecture:** Two pure/small economy changes (`slot_pay`, `sp_econ`) make the stake variable, unit-tested where deps allow. The screen rewrite lives entirely in `slot.lua` — one cohesive file with shared subpixel-canvas state — so it is built **inline** and gets a whole-file code-review pass (per CLAUDE.md's SDD carve-out), not per-task TDD.

**Tech Stack:** Lua 5.1 / LuaJIT (tests), CC:Tweaked (runtime), `src/lib/subpixel.lua` teletext canvas.

## Global Constraints

- Target monitor is **1×2 @ scale 0.5 = 15 cols × 24 rows** (30×72 subpixels). Layout is hardcoded to this.
- **All text white** this pass; non-white colours are out of scope (owner: colour comes later).
- **Reuse verbatim:** `GRAD` / `updateGradient` gradient, `bulb()`, `drawSpriteClipped`, `drawReel`, window+`setVisible` flush, palette save/restore.
- Minecraft levers are **binary 0/15** — rising-edge guard is `>= LEVEL` fire / `< LEVEL` re-arm.
- `slot_pay.eval` must stay **back-compatible**: `eval({r1,r2,r3})` with no stake still returns the old values (existing `test/test_slot_pay.lua` must keep passing).
- Deploy loop: nothing lands in-world until merge+push + in-game `update slot`. In-world verify is post-merge.
- Run tests with `luajit test/test_<name>.lua`; syntax-check any file with `luajit -bl <file> >/dev/null`.

```lua
-- slot_pay.lua — the slot's payout script (pure; unit-tested). The tiny per-game piece the
-- economy gateway needs: a stake ladder and eval(result, stake) -> payout.
-- result = { r1, r2, r3 } symbol indices: 1=seven 2=cherry 3=bell 4=bar.
local STAKE  = 10                                    -- default/back-compat stake
local STAKES = { 10, 25, 100 }                       -- selectable ladder (slot v3)
local MULT   = { [1] = 25, [2] = 3, [3] = 5, [4] = 8 }   -- seven(jackpot) cherry bell bar

return {
  STAKE  = STAKE,
  STAKES = STAKES,
  MULT   = MULT,
  eval = function(result, stake)
    stake = stake or STAKE
    local a, b, c = result[1], result[2], result[3]
    if not (a == b and b == c) then return 0 end
    return stake * (MULT[a] or 0)
  end,
}
```

```lua
  function self.tryBet(stake)
    self.denied = false
    if not self.player then self.round = "free"; return "free" end
    local st = stake or self.pay.STAKE
    local ok, bal = wallet.bet(self.player, st)
    if ok then
      self.balance = bal; card.writeMirror(bal)
      self.round = "staked"; self.stakedId = self.player; self.stakedStake = st

    end

    self.denied = true; self.round = nil
    return "deny"
  end
```

- [ ] **Step 2: Replace `topLayout` with fixed 15×24 cell-row bands.** The canvas is `cv.w=30, cv.h=72`. Define helpers `R(row)` → top subpixel of a cell row = `(row-1)*3+1`, and hardcode the bands from the spec table. Return a layout table exposing: `headerRow` (text rows 2–4, use row 3 centre), `winLabelRow` (6–7), `winAmtRow` (8–9), `topBarY`=`R(9)`, `topBarH`=`2*3` (rows 9–10), `bulbRowY`=`R(11)`, `celebY`=`R(11)`, `viewTop`=`R(15)`, `viewBot`=`R(17)+2` (rows 15–17), `botBarY`=`R(9)` region… concretely:

```lua
local function topLayout(cv)
  local function R(row) return (row - 1) * 3 + 1 end   -- top subpixel of a cell row
  -- reels: 3 sprites (8 wide) spread across cols 2..14 (subpx 3..28)
  local zoneX, zoneW = 3, 26
  local gap = math.floor((zoneW - 3 * SYM_W) / 2); if gap < 0 then gap = 0 end
  local startX = zoneX + math.floor((zoneW - (3 * SYM_W + 2 * gap)) / 2)
  local xs = {}
  for i = 1, 3 do xs[i] = startX + (i - 1) * (SYM_W + gap) end
  return {
    xs = xs,
    headerY   = R(3),               -- <id>: <bal> MB   (cell rows 2-4, centre line)
    winLblY   = R(6),               -- "WIN:"           (rows 6-7)
    winAmtY   = R(8),               -- win amount        (rows 8-9)
    topBarY   = R(9),  topBarH = 6, -- red frame bar     (rows 9-10)
    bulbRowY  = R(11),              -- full-width bulb row (row 11)
    celebY    = R(11),              -- celebration zone   (rows 11-12)
    viewTop   = R(15), viewBot = R(17) + 2, -- reel viewport (rows 15-17)
    playY     = R(18),              -- blue play area top (rows 18-20)
    orangeBarY= R(21),              -- orange bulb bar    (row 21)
    redBarY   = R(22),              -- red bar            (row 22)
    stakeY    = R(23),              -- stake labels + banner (rows 23-24)
    sideBulbTop = R(12), sideBulbBot = R(21), -- cols 1 & 15 vertical bulbs
  }
end
```

- [ ] **Step 3: Rewrite `drawTop` to paint the mockup bands.** Keep the gradient fill (bands over the whole canvas), then paint: red top frame bar at `L.topBarY` (height `L.topBarH`), the reel viewport reels via `drawReel` clipped to `[L.viewTop, L.viewBot]`, the orange bulb bar row, the red bar row, and bulbs. Reuse `bulb()` for: the full-width bulb row (`L.bulbRowY`), the orange bar (`L.orangeBarY`), and the vertical side lanes (cols 1 & 15 between `L.sideBulbTop..L.sideBulbBot`). Draw order: gradient → viewport reels → frame bars → bars → bulbs → (banner text is a `drawTopFrame` overlay). Concretely:

```lua
local function drawTop(cv, reels, bulbTick, result, stakeIdx)
  local L = topLayout(cv)
  local bandH = math.ceil(cv.h / #GRAD)
  for b = 1, #GRAD do cv:fillRect(1, 1 + (b - 1) * bandH, cv.w, bandH, GRAD[b]) end
  -- reels in the single-symbol viewport
  for i = 1, 3 do drawReel(cv, L.xs[i], reels[i], L) end
  -- red top frame bar (flash gold on a win)
  local barCol = (result == "win" and bulbTick % 2 == 0) and YELLOW or RED
  cv:fillRect(1, L.topBarY, cv.w, L.topBarH, barCol)
  -- bars near the bottom
  cv:fillRect(1, L.orangeBarY, cv.w, 3, RED)   -- bulb bar base
  cv:fillRect(1, L.redBarY,    cv.w, 3, RED)
  -- full-width bulb row + orange bulb bar
  for x = 2, cv.w - 2, 4 do
    bulb(cv, x, L.bulbRowY,  math.floor(x / 4), bulbTick, result)
    bulb(cv, x, L.orangeBarY, math.floor(x / 4), bulbTick, result)
  end
  -- vertical side bulbs, cols 1 and 15 (subpx x=1 and x=cv.w-1)
  for y = L.sideBulbTop, L.sideBulbBot, 4 do
    bulb(cv, 1, y, math.floor(y / 4), bulbTick, result)
    bulb(cv, cv.w - 1, y, math.floor(y / 4), bulbTick, result)
  end
  -- celebration flash on a win (first pass: a simple gold band)
  if result == "win" and bulbTick % 2 == 0 then
    cv:fillRect(3, L.celebY, cv.w - 4, 3, YELLOW)
  end
  cv:render()
end
```

Note `drawReel` uses `L.paylineY`; since the viewport is one symbol tall, set the payline to the viewport top. Add to the layout return: `paylineY = R(15)`. Update `drawReel`'s call site is unchanged (it reads `L.paylineY`, `L.viewTop`, `L.viewBot`).

- [ ] **Step 4: Rewrite `drawTopFrame` text overlays.** Header (id/balance), `WIN:` label + amount, stake row with the selected stake highlighted, and the WIN/LOSE banner over the stake band. All text white; text-cell bg set to a gradient palette slot so it rides the gradient. Signature gains `stakeIdx`:

```lua
local function drawTopFrame(reels, bulbTick, result, attract, status, stakeIdx)
  topWin.setVisible(false)
  drawTop(topCv, reels, bulbTick, result, stakeIdx)
  local L = topLayout(topCv)
  local function cellRow(spY) return math.floor((spY - 1) / 3) + 1 end   -- subpx y -> cell row
  local function centreWrite(text, cy, bgSlot)
    topWin.setTextColor(WHITE); topWin.setBackgroundColor(bgSlot)
    topWin.setCursorPos(math.floor((tw - #text) / 2) + 1, cy)
    topWin.write(text)
  end
  -- header: <id>: <bal> MB, or FREE PLAY
  if status then
    local hdr
    if status.denied then hdr = "INSUFFICIENT"
    elseif status.player then hdr = ("%s: %d MB"):format(status.player, status.balance or 0)
    else hdr = "FREE PLAY" end
    centreWrite(hdr, cellRow(L.headerY), GRAD[1])
  end
  -- WIN: label + amount (show the last win)
  centreWrite("WIN:", cellRow(L.winLblY), GRAD[1])
  centreWrite(("%d"):format((status and status.lastWin) or 0), cellRow(L.winAmtY), GRAD[1])
  -- stake row: $10  $25  $100, selected highlighted
  local stakeCols = { 2, 7, 12 }   -- mockup cols for the three labels
  for i = 1, #STAKES do
    local lbl = ("$%d"):format(STAKES[i])
    local sel = (i == stakeIdx)
    topWin.setTextColor(WHITE)
    topWin.setBackgroundColor(sel and WHITE or GRAD[1])
    if sel then topWin.setTextColor(BLACK) end   -- invert selected for legibility (still monochrome)
    topWin.setCursorPos(stakeCols[i], cellRow(L.stakeY))
    topWin.write(lbl)
  end
  -- WIN/LOSE banner over the stake band during a result
  if result == "win" or result == "lose" then
    local label = (result == "win") and "WIN!" or "LOSE"
    topWin.setTextColor(WHITE)
    topWin.setBackgroundColor(result == "win" and GREEN or RED)
    topWin.setCursorPos(math.floor((tw - #label) / 2) + 1, cellRow(L.stakeY) + 1)
    topWin.write(label)
  end
  topWin.setVisible(true)
end
```

(`BLACK` for the selected-stake invert is already a local; keeping the selected chip white-bg/black-fg stays within the monochrome-first-pass intent while reading as "selected".)
