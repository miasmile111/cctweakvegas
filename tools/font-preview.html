pixelfont — glyph specimen & advert screens

This page is a transcription of src/lib/pixelfont.lua's glyph tables. It is a review surface only — not in the src/ deploy loop, and it renders nothing the game reads. If you redraw a glyph here you must carry the change back into the Lua by hand; the JS BIG table below is a verbatim paste of the Lua rows so a diff between them stays readable.

Panel 1 — Specimen

On-pixels white, off-pixels dark grey (so a glyph's box is visible). Each glyph is shown at 1× and 2×, labelled with its width in subpixels. A subpixel = one 8-px block here; on the real monitor a subpixel is ~square.

Panel 2 — Advert screens (left: raw subpixels · right: encodeCell monitor-truth)

The right panel of each pair collapses every 2×3 cell to the 2 colours a real monitor can show (fg + bg), via a port of subpixel.lua's encodeCell — that is what actually renders in-world. The cage's rate table is native cell-text, drawn as an HTML overlay because native write is not subpixel and not subject to encodeCell.
